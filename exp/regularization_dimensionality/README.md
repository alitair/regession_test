# Regularization Dimensionality Experiment

## Intent

This experiment uses the same synthetic data definition and the same shared random model initialization as `exp/loss_invariance`, but it asks a different question:

Can the dimensionality of the data be recovered by training the overparameterized model with different regularizers and then simply counting the non-zero coefficients?

The ground-truth data are still generated by

`y = m1 * x + m2 * x^2`

so the true data dimensionality is `2`. By default Gaussian noise with standard deviation `0.1` is added to the observations.

The fitted model is still overparameterized:

`y = sum(a_n * x + b_n * x^2)`.

With the default configuration `n_terms = 6`, the model has `12` parameters.

## Method

The experiment does the following.

1. Generate the same synthetic quadratic dataset used in `loss_invariance`.
2. Sample the same style of random initial parameter vector.
3. Train that same initialization in three ways:
   - no regularization
   - lasso regularization
   - ridge regularization
4. Count how many learned coefficients remain non-zero above a small threshold.
5. Treat that raw non-zero count as a dimensionality estimate.

This is intentionally different from the Hessian-rank method in `loss_invariance`. Here the estimate is based on raw sparsity in parameter space rather than curvature of the loss.

## CLI

Run the experiment with:

`uv run python exp/regularization_dimensionality/run.py`

Useful flags:

- `--data_root`: accepted for repository-wide consistency. Resolution order is `--data_root`, then `./data`.
- `--results_root`: base directory for outputs. Defaults to `results`.
- `--run_name`: results subdirectory name. Defaults to `regularization_dimensionality`.
- `--n_samples`: number of synthetic samples.
- `--n_terms`: number of duplicated linear and quadratic terms.
- `--m1`, `--m2`: ground-truth coefficients.
- `--noise_std`: additive Gaussian noise level.
- `--seed`: random seed.
- `--epochs`: number of gradient steps.
- `--learning_rate`: optimizer step size.
- `--lasso_lambda`: lasso penalty strength.
- `--ridge_lambda`: ridge penalty strength.
- `--zero_threshold`: threshold used when deciding whether a coefficient is non-zero.
- `--history_stride`: history sampling stride for CSV output.

## Interpretation

The comparison is meant to show the difference between fitting the data well and recovering a sparse representation.

- No regularization can fit the data, but it usually keeps many redundant coefficients active.
- Ridge regularization also fits the data, but it spreads weight across redundant coefficients instead of removing them.
- Lasso regularization tends to keep a sparse solution and therefore produces the smallest raw non-zero parameter count.

Because the estimate is just the raw non-zero coefficient count, it generally overestimates the true dimensionality `2` in this duplicated-feature model.

## Outputs

Each run writes a subdirectory under `results/`.

Main files:

- `regularization_dimensionality.png`: main comparison figure.
- `summary.json`: scalar summary for each training method.
- `training_history.csv`: loss and objective history.
- `final_coefficients.csv`: learned coefficients and sparsity counts.
- `README.md`: run-specific result summary.

## Figure Guide

The figure is organized into six panels.

1. Ground-truth data.
   Shows the synthetic dataset and the true generating curve.

2. Shared random start.
   Shows the untrained model before optimization. All three methods start from this same parameter vector.

3. Final fitted curves.
   Compares the three trained models against the true curve.

4. Training loss.
   Shows how quickly each method reduces the data loss.

5. Final coefficient magnitudes.
   Shows which raw coefficients remain active after training and where the non-zero threshold is applied.

6. Dimensionality estimate.
   Shows the raw non-zero coefficient count for each method and compares it with the true value `2`.

## Notes

- This experiment is synthetic and uses only the Python standard library.
- It complements `exp/loss_invariance` rather than replacing it.
- This experiment intentionally uses a crude dimensionality estimate so its failure mode is visible.
