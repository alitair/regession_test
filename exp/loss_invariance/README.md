# Loss Invariance Experiment

## Intent

This experiment studies a simple question: can the set of parameter values that preserve the same loss reveal the true dimensionality of the underlying data?

The toy problem is deliberately small so that the symmetry is easy to see. The ground-truth data are generated by

`y = m1 * x + m2 * x^2`

while the model is overparameterized:

`y = sum(a_n * x + b_n * x^2)`.

Although the model has many parameters, its output depends only on two aggregate quantities:

- `A = sum(a_n)`
- `B = sum(b_n)`

That means many different parameter vectors produce the same function and therefore the same loss.

## CLI

Run the experiment with:

`uv run python exp/loss_invariance/run.py`

Useful flags:

- `--data_root`: accepted for repository-wide CLI consistency. Resolution order is `--data_root`, then `./data`.
- `--results_root`: base directory for outputs. Defaults to `results`.
- `--run_name`: results subdirectory name. Defaults to `loss_invariance_analysis`.
- `--n_samples`: number of synthetic samples.
- `--n_terms`: number of duplicated linear and quadratic model terms.
- `--m1`, `--m2`: ground-truth coefficients.
- `--noise_std`: additive Gaussian noise level.
- `--seed`: random seed.
- `--direction_steps`: number of samples used when scanning loss along parameter directions.
- `--equivalent_samples`: number of equal-loss parameter samples to generate.

Example:

`uv run python exp/loss_invariance/run.py --run_name noisy_case --noise_std 0.1 --equivalent_samples 120`

## Pipeline

The code follows this order.

1. Generate the ground-truth dataset.
   The script builds `x` values on a fixed interval and evaluates the quadratic ground-truth function.

2. Draw an untrained random model.
   Random coefficients are sampled for the overparameterized model to show what the function looks like before fitting.

3. Fit the overparameterized model.
   The fit is solved in aggregate form. Even though the parameter vector is long, only `A = sum(a_n)` and `B = sum(b_n)` determine the predictions.

4. Generate equal-loss parameter samples.
   Starting from one fitted parameter vector `theta*`, the script draws perturbations that satisfy:
   `sum(delta_a) = 0` and `sum(delta_b) = 0`.
   Adding those perturbations changes individual parameters, but it does not change `A` or `B`. Because the prediction is unchanged, the loss is unchanged as well.

5. Recover the active dimensionality.
   The script builds the Hessian of the loss at the fitted solution. In this toy model the Hessian has exactly two non-zero directions, corresponding to the two aggregate coefficients that affect the output.

6. Write explanatory outputs.
   The script writes a process figure, machine-readable summaries, CSV tables, timing information, and a run-specific README under `results/<run_name>/`.

## Method Interpretation

The central idea is:

- Many parameter vectors are equivalent because the model output ignores most directions in parameter space.
- The equal-loss set is therefore high-dimensional in parameter space.
- Locally, the loss only changes along the active directions of the Hessian.
- Counting those active directions recovers the effective dimensionality seen by the loss.

In this experiment, that count is 2, which matches the true generative structure of the data.

## Outputs

Each run writes a subdirectory under `results/`.

Main files:

- `loss_invariance_process.png`: the main explanatory figure.
- `summary.json`: scalar summary and artifact paths.
- `timings.csv`: runtime by major stage.
- `equivalent_solutions.csv`: sampled parameter vectors with the same loss.
- `direction_scan.csv`: loss sampled along one active direction and one null direction.
- `README.md`: run-specific summary and conclusions.

## How To Read The Figure

The process figure is organized as a visual narrative.

1. Ground-truth data.
   Shows the synthetic dataset and the true generating curve.

2. Random untrained model.
   Shows that arbitrary parameters do not generally explain the data.

3. Ground truth versus fitted model.
   Shows that the fitted model can match the target curve despite having many more parameters than the data dimension.

4. Equal-loss sample generation.
   Explains how to perturb parameters while preserving the aggregate coefficients and therefore the loss.

5. Dimensionality recovery.
   Shows the Hessian spectrum and the rule used to count active directions.

6. Parameter-space versus function-space view.
   Shows that equal-loss samples can spread out in null directions while collapsing to a single point in the aggregate coefficient space that controls the function.

## File Layout

The code for this experiment is kept together in this directory.

- `run.py`: main entry point and output writer.
- `analysis.py`: data generation, fitting, equal-loss sampling, and dimensionality calculations.
- `png_dashboard.py`: low-level PNG rendering for the explanatory figure.

## Notes

- The experiment is synthetic and does not require external input data.
- `--data_root` is still accepted so that the CLI matches the repository-wide convention.
- The implementation uses only the Python standard library.
